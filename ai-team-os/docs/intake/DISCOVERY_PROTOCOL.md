# 🔍 DISCOVERY PROTOCOL — Primera Sesión de Proyecto
> Ejecutar SIEMPRE en la primera sesión de un proyecto nuevo.
> El objetivo: entender el negocio ANTES de escribir una línea de código.

---

## CUÁNDO EJECUTAR

- Primera sesión de un proyecto nuevo
- Cuando el CEO dice "NUEVO PROYECTO"
- Cuando se retoma un proyecto pausado más de 3 meses

---

## REGLAS DEL DISCOVERY

1. El Orquestrador hace las preguntas — no las hace todas de golpe
2. Máximo 3 preguntas por mensaje
3. Escuchar activamente — las respuestas generan nuevas preguntas
4. NO proponer tecnología hasta terminar el discovery
5. NO estimar tiempos hasta entender el alcance completo
6. Al final → presentar el resumen y pedir confirmación del CEO

---

## PREGUNTAS OBLIGATORIAS (en este orden)

### BLOQUE 1 — El Negocio
1. ¿Qué problema resuelve este producto? ¿Para quién?
2. ¿Ya existe algo similar en el mercado? ¿Qué hace diferente este?
3. ¿Cómo gana dinero este producto? (modelo de negocio)

### BLOQUE 2 — El Usuario
4. ¿Quién es el usuario típico? (edad, contexto, nivel técnico)
5. ¿Cómo llega el usuario al producto hoy sin él? ¿Qué hace ahora?
6. ¿Qué tiene que pasar para que el usuario pague?

### BLOQUE 3 — El Producto
7. ¿Cuáles son las 3 funciones más importantes del MVP?
8. ¿Qué NO debe tener el MVP? (definir límites)
9. ¿Hay alguna funcionalidad sin la cual el producto no tiene sentido?

### BLOQUE 4 — El Contexto Técnico
10. ¿Hay integraciones con otros sistemas? (pagos, WhatsApp, ERP, etc.)
11. ¿Hay datos existentes que migrar o un sistema que reemplazar?
12. ¿Quién va a mantener el sistema después de entregado?

### BLOQUE 5 — Tiempo y Recursos
13. ¿Cuándo necesitas tener el MVP listo?
14. ¿Cuál es el presupuesto aproximado?
15. ¿Hay un desarrollador externo o todo es con este equipo?

---

## PREGUNTAS ADICIONALES (según el tipo de proyecto)

### Si tiene pagos:
- ¿En qué países? ¿Solo México o internacional?
- ¿Suscripción, pago único, o marketplace?
- ¿El cliente ya tiene cuenta con algún procesador de pagos?

### Si tiene usuarios con roles:
- ¿Cuántos tipos de usuarios hay? ¿Qué puede hacer cada uno?
- ¿Hay aprobaciones o flujos de autorización?

### Si tiene datos sensibles:
- ¿Qué tan sensibles son los datos? (médicos, financieros, menores de edad)
- ¿Hay requerimientos de privacidad especiales?
